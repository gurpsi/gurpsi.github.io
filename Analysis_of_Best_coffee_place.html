  • Documenting and Visualising the top trends in the data.

API Details:¶

Yelp API:¶

Following are the parameters set for fetching data from Yelp using its API:

City = Dublin

API Endpoint for Business ID's: https://api.yelp.com/v3/businesses/search.json

API Endpoint for Reviews: https://api.yelp.com/v3/businesses/{id}/reviews.json

Secret API Key: Can't Share

API Limit-day: 25000

  • The response contains all the business Id's and then using these business Id's reviews related data are fetched.
  • This data is Filtered and Feature are selected to perform the analysis.
  • Max of 1000 business-ids can be fetched from the given parameters.
  • Max 3 reviews per business id is returned from the API.

Normalising Data:¶

Normalization is often done on data so as to remove amplitude variation and only focusing on the underlying distribution.

Here the data is normalised in range of 0 and 1, by dividing the 'number of reviews' by the 'max number of reviews'. As this will differentiate the most popular shop from the rest of the shops, and all the values in the data set will be positive values.

Whereas normalising using mean will result in negative values: z(a) = (x(a) mean(x(a))) / std(x(a))

Another way I could do is using following formula for normalisation: (x-min) / (max-min)

Calculating the Polarity and the Subjectivity of the reviews:¶

Polarity:¶

  • It simply means emotions expressed in a sentence. It is a measure of the negativity, the neutralness, or the positivity of the text.

Subjectivity:¶

  • Subjective sentence expresses some personal feelings, views, or beliefs.

Range:¶

  • Polarity: -1 to 1, i.e. negative to positive emotions.
  • Subjectivity: value from 0 to 1 measuring the subjectivness of the text. 0 is objective, 1 is subjective.
  • Calculating the Polarity and Subjectivity and storing it in the dataframe.

Based on the Polarity score, Reviews are categorised:¶

* Positive: If the polarity score is greater than .35 then the statement is considered as positive.¶

* Negative: If the polarity score is smaller than than -.35 then the statement is considered as negative.¶

* Neutral: If the polarity score is between -.34 and .34 then the statement is considered as neutral.¶

  • This is done as the 'range of polarity' varies from -1 to 1.
  • For 1 being very happy or Positive
  • And -1 being very bad or Negative.

Hence we have i.e. 2 points (-1 to 1) and dividing it by 3 features namely Positive, Neutral and Negative. Thereby the polarity is being further categorised into 3 features.

And 3 are columns are added in the dataframe as Positive, Negative and Neutral based on the score.¶

In [20]:
df_reviews['Positive'] = np.where(df_reviews.Polarity >= .35,1,0)
